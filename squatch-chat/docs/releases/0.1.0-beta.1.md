# Campfire 0.1.0-beta.1

## Release status and scope

This is a release candidate for an invited, small-community, self-hosted beta. It is not a general-availability release or a production SLA. The target is the free AGPL Campfire Community edition running from source or the PostgreSQL Compose stack.

Campfire Cloud billing and a paid managed service are outside this beta. No Windows portable or installer artifact is part of the beta until a candidate-specific artifact is built, signed, and smoke-tested on a clean machine.

## Included product work

The candidate contains the Discord-shaped chat, community, moderation, private attachment, realtime, and peer-to-peer voice foundation described in the [README](../../README.md). It also contains the ten Campfire features tracked in the [gap analysis](../GAP_ANALYSIS.md): managed invites, personal blocks, voice notes, purpose-driven voice rooms, Pass the Lantern, leave-no-trace rooms, Camp Journal, Camp Votes, Camp Gatherings, and Offshoots.

Authorization, session, billing-boundary, attachment, proxy, and production-configuration hardening is included in source. Passing source tests does not replace the exact-candidate staging, security, media, upgrade, and recovery evidence listed below.

## Database changes and operator upgrade actions

This candidate includes the current SQLite schema synchronization path and PostgreSQL migration history. Compatibility with a realistic previous-version data set remains a release gate, so make a restorable pre-upgrade snapshot before changing source or schema.

1. Schedule a maintenance window and stop writes to the instance.
2. Back up the database, `.env`, public uploads, avatars, and private uploads as one recovery set. Encrypt it, copy it off host, and verify it can be restored before proceeding. See [DEPLOY.md](../DEPLOY.md#backups).
3. Record the currently deployed commit or tag, then check out the exact beta candidate and use Node.js 22 LTS.
4. Run `npm ci`. On SQLite this runs schema synchronization; do not skip it. On PostgreSQL also run `npm run db:migrate` before restarting.
5. Run `npm run release:check`, `npm run db:check`, and `npm run build`.
6. Restart the service with the custom `npm run host` runtime (normally through the service manager), then complete health, login, message, attachment, restart, and voice smoke tests.

Public instances must retain their explicit Community edition, strong JWT secret, HTTPS, strict CORS allowlist, exact trusted-proxy hop count, persistent storage, working TURN configuration, monitoring, and recovery-email settings where password recovery is expected.

## Known limitations

- Voice is browser WebRTC peer-to-peer mesh. Rooms above roughly six active participants can become expensive in client CPU and upload bandwidth; an SFU is not shipped.
- Pass the Lantern and Offshoot membership are process-local and reset on server restart. Offshoot audio routing is a client-side conversation aid, not a privacy or cryptographic boundary.
- Stored messages and media are readable by the server operator; stored text is not end-to-end encrypted. Legacy public media URLs remain subject to the boundaries documented in [SECURITY.md](../SECURITY.md).
- Exact-candidate HTTPS, reverse-proxy, strict-CORS, TURN-across-networks, browser/media, Docker persistence, upgrade, and backup/restore acceptance are not yet recorded.
- No `0.1.0-beta.1` desktop binary has been validated or published. Earlier `0.0.3` local binaries are not beta artifacts, and the current desktop path remains unsigned and subject to the desktop gates in the [release checklist](../RELEASE_CHECKLIST.md#7-windows-portable-and-installer-workflow).
- Campfire Cloud, payments, tenant-isolation go-live, abuse operations, support, and on-call processes are outside this beta.

## Rollback

Rollback requires the pre-upgrade recovery set; a code-only downgrade is unsafe unless schema compatibility has been proven for the exact versions.

1. Stop the Campfire service and prevent new writes.
2. Restore the previously recorded source commit or tag and reinstall/build its locked dependencies.
3. Restore the database, all media directories, and `.env` together from the pre-upgrade snapshot.
4. Start with `npm run host` and verify health, authentication, representative messages and attachments, restart persistence, and voice signaling.

A failed migration and rollback rehearsal against realistic data is still required before this candidate can be tagged.
